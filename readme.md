## Overview

PACE is a benchmarking tool for evaluating accessibility in HTML generated by large language models.

It supports controlled comparisons across:

- **models** — the LLMs being tested
- **system instructions** — setup messages given before each task
- **form component types** — HTML form elements such as text inputs, radio groups, and buttons
- **request variants** — different phrasings for the same component request

Each generated output is evaluated with two complementary layers:

1. **axe-core testing** in `bench/axe_runner.py`
2. **schema-based scoring** in `bench/scoring.py`

These layers compare automated accessibility findings and component-specific structural checks respecitively.

## Quick Start

Clone the repository:

```bash
git clone https://github.com/momentine/pace.git
cd pace
```

Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

For Windows PowerShell:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
pip install streamlit pandas numpy altair requests python-dotenv beautifulsoup4 lxml playwright
playwright install
npm install axe-core
```

PACE expects axe-core here:

```text
node_modules/axe-core/axe.min.js
```

## Model and API Setup

PACE currently uses the OpenRouter API for model calls.

Create a `.env` file in the project root:

```env
OPENROUTER_API_KEY=your_api_key_here
```

Refresh the OpenRouter model list:

```bash
python scripts/fetch_openrouter_models.py
```

This updates:

```text
data/models.json
```

The Streamlit app reads `data/models.json` to populate available models in the interface.

Expected output:

```text
Saved 123 models -> data/models.json
```

Run this script again whenever you want to refresh available OpenRouter models.

Azure API support is being embedded, but OpenRouter is the current default path.

## Run the App

```bash
streamlit run app.py
```

Streamlit usually starts at:

```text
http://localhost:8501
```

Open that address in your browser.

## Running a Benchmark

1. Open the Streamlit app.
2. Go to **Settings**.
3. Inspect or edit the registries.
4. Go to **Run**.
5. Select models, system instructions, request variants, component tasks, and repetitions.
6. Click **Run**.

Run time depends on the selected models, tasks, and repetitions.

## Results

Each benchmark run is saved under:

```text
output/_runs/
```

A run directory may contain:

```text
results.csv
per_check.csv
generated HTML files
```

`results.csv` stores trial-level results.

`per_check.csv` stores detailed accessibility check results.

## Accessibility Notes

PACE studies accessibility, so this repository should remain usable with assistive technologies.

Please keep headings descriptive, give images meaningful alt text, use language tags for code blocks, and report accessibility barriers through repository issues or direct contact with the maintainers.

## Scope

PACE currently evaluates AI-generated HTML form components. The tool can be adapted to other interface outputs by changing component definitions, scoring rules, and evaluation pipelines.
